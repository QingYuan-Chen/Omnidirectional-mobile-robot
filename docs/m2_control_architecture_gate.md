# M2 电机控制架构决策门

## 目的

M2 在实现最终单电机闭环前，先用可复现的硬件测试和量化指标决定控制架构。参考工程只提供引脚、外设和初始软件框架，不作为控制频率或控制器选型依据。

本决策门撤销以下冻结结论：

- 速度环固定为 100 Hz；
- 速度 PI + 前馈已经是最终电机控制器；
- 10 ms 编码器计数加普通低通足以覆盖全部速度范围。

在决策门关闭前，LADRC 是基准控制器，PI + 电压/死区前馈和 PI + DOB 是对比候选；三者必须使用相同输入、限幅、安全状态机和测试数据进行比较。基准只规定首个实现和评测参照，不代表已经选定最终方案。

## 已确认硬件边界

- MCU 为 STM32F407VET6，当前 HSE 8 MHz、SYSCLK/HCLK 168 MHz。
- 实际电机已确认为塔克 MC520P30_12V：12 V、1:30 减速比、1024 PPR 磁编码器版本。原厂额定/空载转速分别为 290 ± 20 rpm 和 360 ± 20 rpm，额定/堵转电流分别为 0.3 A 和 3.2 A，额定/堵转转矩分别为 1.5 kg·cm 和 4.5 kg·cm。
- 电池已确认为 LB350E：3 串 21700 锂电池、5000 mAh、标称 11.1 V、满电 12.6 V、推荐充电电流 2 A、最大持续放电电流 8 A、最大瞬时放电电流 16 A，带过充、过流、过放、短路和过压保护。厂家参数图未给出 BMS 具体欠压/过流动作值及允许瞬时持续时间。
- 整车空载质量为 2.2 kg，最大额外载荷约 10 kg，按此定义最大总质量约 12.2 kg；车轮直径为 77 mm，底盘和主要机械结构为金属材质。电机安装和出线方式与参考底盘及源码约定一致，当前先沿用参考符号映射。
- AT8236 驱动四路有刷直流电机，当前 PWM 约 20 kHz。
- 原理图中每路 AT8236 的 `ISEN` 使用 0.15 Ω 电阻，`VREF` 固定为 3.3 V；按芯片公式，硬件电流斩波阈值约为 2.2 A。
- `ISEN` 未连接到 MCU ADC；当前 ADC2_IN10 只采集电池电压，因此软件没有真实电流/转矩反馈。
- 原厂将磁编码器标为 1024 PPR，并在 1:30 示例中按四倍频给出每轮 122880 计数。2026-07-18 MA 车轮人工向前三整圈的完整起止平台累计 -371783 counts，平均绝对值 123927.7 counts/rev，与 122880 偏差约 0.85%，据此关闭四倍频定义并排除 30720。

不得使用估算电流伪装真实电流反馈。如果最终需要电流内环，必须单独评审硬件改版、模拟前端、同步采样、保护和标定方案。

## 时钟与执行域

| 域 | 当前值 | 决策门候选 | 要求 |
|---|---:|---:|---|
| MCU SYSCLK/HCLK | 168 MHz | 保留 168 MHz | 不通过修改主频调控制周期；用 MCO/测量手段确认实际时钟 |
| PWM 载波 | 20 kHz | 20 kHz 起测 | 根据声噪、驱动温升、电流纹波和分辨率决定，最终需测量而非只信寄存器 |
| 电流环 | 无 | 10-20 kHz，仅硬件改版后 | 与 PWM/ADC 触发同步，禁止在无电流反馈时建立伪电流环 |
| 速度采样与控制 | TIM7 候选 1 kHz 软件基线，待板测 | 1 kHz 起测 | 最终由机械带宽、总延迟、测速噪声和离散稳定裕度决定 |
| 机体运动学/底盘控制 | 未实现 | 100-200 Hz 起测 | 与电机速度环解耦，只发布轮速目标 |
| FreeRTOS tick | 1 kHz | 保持系统调度用途 | 不作为最终速度环的精确定时基准 |

资源审查已经确认 TIM6 被 HAL 1 kHz 时间基准占用，因此确定性速度环使用当前空闲的 TIM7 更新事件。候选参数为 84 MHz 定时器时钟、`PSC = 83`、`ARR = 999` 和数值中断优先级 5。详细实施边界见 `m2_g1_control_resource_audit.md`。ISR 只允许：

1. 记录硬件时间戳；
2. 快照编码器计数；
3. 向 `controlTask` 发送直接任务通知；
4. 记录超期/丢通知计数。

控制计算、电机状态机和遥测不得放进定时器 ISR。`controlTask` 必须记录实际 `dt`、开始/结束周期计数、WCET、唤醒抖动和截止期违约计数。定时器选择、触发链和优先级在修改 `.ioc` 前单独检查资源冲突。

方向不属于控制器调参项。若首次低功率开环斜坡发现指令方向和编码器速度符号不一致，只能在 BSP/配置层修改电机或编码器符号；闭环使能前必须保证正向控制量产生正向速度反馈，避免把负反馈变成正反馈。

## 决策门步骤

### G0：硬件与电机身份确认

- `[已确认]` 实际电机为 MC520P30_12V，额定电压、转速、电流、转矩、减速比和编码器购买版本已有[原厂产品页](https://www.xtark.cn/product/show.php?itemid=25)证据。
- `[已确认]` 电池型号、电芯串数、容量、电压和标称放电能力，以及整车质量、最大载荷、轮径、机械材质和参考安装方式已有用户提供的规格图及实物信息。
- `[待确认]` 向厂家确认允许温升/绝缘等级；原厂图中反电动势和转矩系数缺少单位，暂不直接用于建模。
- `[已确认]` MA 三圈完整平台实测确认轮端分辨率采用 122880 counts/rev；软件正向对应物理向后和编码器正计数，软件反向对应物理向前和编码器负计数，其余三轮留到 M4 联调。
- `[待板测]` 测量空载和分级载荷下的四轮静态承重、带载电池压降、BMS 欠压/过流动作、单路和四路同时启动行为；不得把“金属材质、耐温高”替代为量化温度上限。
- 测量并记录 AT8236 快衰减/慢衰减控制方式、正反转逻辑、Brake/Coast 行为和硬件电流斩波边界。

电源边界必须纳入安全设计：四台电机目录堵转电流之和为 12.8 A，高于电池 8 A 持续能力；即使四路均受约 2.2 A 硬件斩波限制，理想总和仍约为 8.8 A，略高于持续能力。该估算不代替实测，G1 必须限制同时启动斜率并记录电池电压和保护动作。

缺少电机型号或关键额定参数时，不允许用参考电机参数代替。

### G1：确定性测量与安全试验台

- 可配置调试串口提供带序号和时间戳的命令/遥测通道；M2 Type-C 板测临时使用 USART2，M5 正式 X-Protocol 前迁回 UART4。
- 控制节拍已改为 TIM7 候选 1 kHz 硬件定时器通知；最终频率仍由 G7 决策。
- 所有试验必须具备 PWM 上限、目标斜坡、命令超时、受控停车、立即 Coast 和人工急停。现有破坏性 Emergency Coast 调用后必须系统复位，不允许串口远程重新使能；可恢复 IMU 降级不得误用该接口。
- 遥测按阶段补齐且只报告真实产生的数据：G1.3 至少包含时间戳、原始编码器计数/增量、PWM 目标值/实际施加值、电池电压、安全状态、WCET、抖动和通信诊断；G3 完成速度估计后追加速度字段；G5 接入控制器后再追加未限幅/已限幅控制量、方向/衰减模式和控制器内部状态。字段尚未由对应阶段产生时不得伪造占位值。

时基验收：连续测试无丢周期；P99 唤醒抖动不超过周期的 5%，最大抖动不超过 10%；最坏 WCET 不超过周期的 25%。未达到时先修时基，不进入控制器比较。

### G2：开环辨识数据集

在安全架空工况先完成正反转测试，再根据风险评估增加落地或受控负载测试：

- 多档电池电压或不同荷电状态；
- 正反转阶跃、斜坡、零点穿越和 Brake/Coast；
- 多档 PWM，包括死区、线性区、饱和区和电流斩波可能介入的区域；
- 每个工况重复至少三次，并记录环境、温升和机械配置。

辨识对象至少比较一阶加纯延迟、二阶加纯延迟模型；报告参数置信区间、残差、自相关和独立验证集误差，不只给一条拟合曲线。

### G3：速度测量方案评审

- 比较固定窗口 M 法、脉冲周期 T 法以及按速度切换/融合的 M/T 方案。
- 在 1 kHz 候选节拍下统计各速度段的计数量化、延迟、方差、零速误判和方向切换行为。
- 普通低通只能作为候选链路的一部分，必须计入相位延迟；不得用过度平滑掩盖低速不可观测问题。

### G4：电流反馈硬件决策

- 评估保留 H60、引出并调理 ISEN，或更换带电流镜/故障输出驱动器三种方案。
- 若改版，明确分流电阻功耗、共模范围、放大器带宽、ADC 触发相位、模拟滤波、过流保护独立性和四路采样预算。
- 输出书面决定：`固定现有硬件` 或 `增加真实电流反馈`。未完成此决定，不冻结最终控制结构。

### G5：统一控制器候选

实现与 HAL 无关的统一控制器接口，并在 host 仿真中使用同一辨识模型和离散周期：

1. 一阶对象 LADRC，使用二阶线性 ESO，作为基准控制器；
2. PI + 电压/死区前馈，作为可解释对比候选；
3. PI + 模型型 DOB，明确名义模型、逆模型实现和 Q 滤波器。

三种控制器共用目标整形、输出限幅、方向逻辑、命令超时和安全状态机。控制器切换必须在停稳状态进行，并重置内部状态，禁止带状态热切换。

### G6：统一实机对比

对每个候选控制器使用相同测试序列、初始条件和约束，至少统计：

- IAE、ITAE、稳态误差、超调、上升/调节时间；
- 负载扰动恢复时间和峰值速度跌落；
- 电池电压变化敏感度；
- 低速速度纹波、零速保持和方向反转；
- 饱和退出、积分/观测器恢复和命令超时停车；
- 控制量 RMS/峰值、驱动温升、WCET、抖动和异常计数；
- 对辨识参数偏差、采样延迟和编码器噪声的 Monte Carlo/扫参结果。

不得只凭单次阶跃波形或主观平滑度选型。

### G7：架构决策记录

最终决策必须记录：

- 选定硬件反馈结构、PWM、速度环和底盘环频率；
- 选定控制器及未选方案的量化原因；
- 速度估计方法、闭环带宽、相位/增益裕度或等价离散鲁棒性证据；
- 全部限幅、复位、故障降级和参数范围；
- 可复现实验数据、分析脚本、固件提交和板上验收记录。

只有 G0-G7 全部关闭后，才能把控制器和速度环频率写入“已确认决策”。

## 当前执行边界

- 当前不修改 168 MHz SYSCLK。
- G1.0 与 TIM7 候选 1 kHz 软件实现已经完成；固定 CCMRAM 电机高速记录器也已通过无动力和带动力完整时序板测。原路线中的 IMU 高速记录模式尚未实现。
- Type-C/USART2 的 COM6 双向通信和 Windows 可复现采集工具已经关闭。当前低频遥测仍是单个 `T,...` 50 Hz 帧，`STAT/IMUQ/RES/EVENT` 分类型遥测、五任务栈水位和最小剩余堆尚未实现或验收。
- 当前处于 G2/G3 动态辨识采集：`±400`、`±600` 和 `+240` 重复性已通过；全新五次 `-240` 原短窗口均值为 8.425 RPM、`CV=8.009%`，达到五次上限仍未关闭。离线诊断显示平台末端 200/300 ms 速度 `CV=3.674%/4.393%`，但启动延迟仍明显离散，当前只形成“稳定运行速度、离散启动过程”的探索性假设。
- 下一动作是执行最多三次的预注册独立验证：固定平台末端 200/300 ms 双窗口、5% CV、10% 最大单次偏离、初始轮位、单向靠近、至少 60 s 冷却和 11.500-11.750 V 预检范围。不得看完结果后选窗口或追加第四次；旧短窗口门继续原样保留。`+840` 仅保留一次单次接受证据，其余 840、模型拟合和控制器实现继续暂停。
- 延期的 M1.3 联合验收、分类型遥测和 IMU 高速模式必须明确排期，不得因电机 G2 已开始而静默写成完成。
- 当前不实现伪电流环。
- 当前把 LADRC 写为基准控制器，把 PI + 前馈和 PI + DOB 写为对比候选，但不将任何一个写成最终方案。
