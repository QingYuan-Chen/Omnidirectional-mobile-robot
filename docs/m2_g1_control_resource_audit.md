# M2 G1 控制资源审查与实施约束

## 文档状态

- 审查阶段：G1.0 资源审查完成，G1.1/G1.2 软件完成并待板测
- 审查日期：2026-07-15
- 适用对象：G1.1 确定性控制节拍、G1.2 MA 安全开环链路、G1.3 UART4 命令与遥测
- 硬件验收：不在 G1.0 内执行；与 G1 软件完成后的单电机板测及延期的 M1.3 一并进行

## 控制器定位

- LADRC 是后续 G5 首个实现和统一评测的基准控制器。
- PI + 电压/死区前馈和 PI + DOB 是使用同一接口、数据集、约束和指标的对比候选。
- 当前实现范围明确采用线性 ADRC（LADRC），不把非线性 ADRC 作为第四种候选；若未来引入，必须另行建立参数、离散化和统一评测门槛。
- “基准控制器”不等于“最终控制器”。G6 实机对比和 G7 决策记录关闭前，不得把任何候选写成最终方案。
- 当前硬件没有 MCU 可见的电机电流反馈，禁止实现伪电流环。

| 资源 | 当前事实 | G1 决策 | 状态 | 验证依据 |
|---|---|---|---|---|
| SYSCLK/HCLK | 168 MHz | 保持不变 | 已冻结 | `.ioc` RCC 配置与 `SystemClock_Config()` |
| APB1 PCLK | 42 MHz | 保持不变 | 已冻结 | APB1 分频为 4 |
| APB1 定时器时钟 | 84 MHz | 作为 TIM7 输入时钟 | 已冻结 | APB1 分频不为 1 时，定时器时钟为 PCLK 的 2 倍 |
| TIM6 | HAL 1 kHz 时间基准，中断优先级 0 | 禁止复用为控制节拍 | 已冻结 | `stm32f4xx_hal_timebase_tim.c` 和 `HAL_TIM_PeriodElapsedCallback()` |
| TIM7 | 已配置并由控制任务启动 | 用作候选 1 kHz 控制节拍 | 软件已实现、待板测 | `.ioc`、`tim.c`、控制时基模块 |
| TIM7 预分频 | `PSC = 83` | `PSC = 83` | 软件已实现、待板测 | `84 MHz / (83 + 1) = 1 MHz` |
| TIM7 自动重装 | `ARR = 999` | `ARR = 999` | 软件已实现、待板测 | `1 MHz / (999 + 1) = 1 kHz` |
| TIM7 IRQ 优先级 | 数值优先级 5，子优先级 0 | 保持 5/0 | 软件已实现 | `configLIBRARY_MAX_SYSCALL_INTERRUPT_PRIORITY = 5` |

TIM7 ISR 允许执行的动作限定为：读取 DWT 时间戳、快照编码器原始计数、更新节拍诊断计数、发送计数型任务通知以及必要的 `portYIELD_FROM_ISR()`。控制计算、电机状态机、ADC、UART、字符串格式化、阻塞式 HAL 调用和浮点日志不得进入 ISR。

## CubeMX 再生成边界

- 规范输入文件为 `firmware/r750_mec_cubemx/R750_MEC_Lower.ioc`。
- `firmware/r750_mec_cubemx/R750_MEC_Lower/R750_MEC_Lower.ioc` 是项目目录内的生成副本，不作为手工编辑入口。
- 使用 `firmware/r750_mec_cubemx/cubemx_generate.txt` 加载规范输入并重新生成。
- TIM7 的时钟、NVIC、初始化调用和 HAL 句柄由 `.ioc` 生成；应用行为放在 `App/`，生成文件只允许在 USER CODE 区保留人工代码。
- 每次生成后必须审查两个 `.ioc`、`tim.h/.c`、`stm32f4xx_it.h/.c`、`main.c` 和构建源文件列表，拒绝与 TIM7 无关的生成差异。
- 工程 `R750_MEC_Lower/CMakeLists.txt` 显式列举 `App` 和 BSP 源文件，不使用自动 glob。G1.1、G1.2、G1.3 新增的每个 `.c` 都必须手动加入 `target_sources()`，并分别通过 Debug 和 Release 构建确认没有漏编。
- `HAL_TIM_PeriodElapsedCallback()` 已由 `Core/Src/main.c` 定义并承担 TIM6 HAL tick；TIM7 只能在该回调的 USER CODE 区分派到应用模块，禁止在 `App` 再定义同名 HAL 回调。
- UART RX 完成和错误回调已经由 `App/Bsp/Src/bsp_uart.c` 统一定义。G1.3 的 TX 完成回调必须合并到同一 BSP 回调分派中，禁止在其他模块重复定义 HAL UART 回调。

## 任务与通知适配

- `controlTask` 保持 High 优先级，G1.1 已删除控制用 `vTaskDelayUntil()`，改为等待 TIM7 的计数型直接任务通知。
- 控制任务必须先登记原生 `TaskHandle_t`，随后才允许启动 TIM7，避免启动窗口内通知无接收者。
- 一次唤醒收到的通知计数大于 1 时记录合并/迟到周期。通知数通过 16 槽带序号环定位对应 ISR 快照，避免后到中断造成通知和单份最新快照错配。
- UIF 折叠造成的硬件漏更新按启动相位累计理论应到更新数与实际 ISR 服务数之差检测；控制任务漏迭代和通知合并使用独立计数，三者不得混写。
- 控制任务继续按低于 1 kHz 的分频节奏上报心跳，避免每周期操作事件标志；安全任务用单次清除操作取得清除前事件位，首次缺失普通 Coast，连续 3 个窗口缺失才锁存并 Emergency Coast。
- DWT CYCCNT 作为硬件周期计数。168 MHz 下 32 位计数约 25.6 s 回绕，所有时间差采用无符号减法，并使用节拍序号辅助主机侧展开。

## 编码器与电机边界

- 四路编码器分别为 TIM2、TIM3、TIM5、TIM4。当前 BSP 原始读取接口返回 `uint16_t`，增量接口内部保存上次计数。
- G1.1 在 ISR 快照四路原始 16 位计数，在控制任务中用模 2^16 的 `int16_t` 差值计算单周期增量，并用 `int64_t` 累计总计数。
- 启用 ISR 快照后，控制任务不得再并行调用带独立历史状态的 `BspEncoder_ReadDelta()`，避免两套基线产生不一致。
- 当前 PWM 绝对满量程为 4200。G1.2 试验硬上限为 840，即满量程的 20%；该上限只能由编译期配置修改，UART 命令不得提高。
- G1.2 只允许 MA 受控输出，MB、MC、MD 强制 Coast。上电默认运动禁止，Safety 首次健康清点后进入未使能状态；任何安全故障都优先于运动命令。
- 目标斜坡初值为每个 1 ms 控制周期最多变化 1 个 PWM 计数；命令超时为 500 ms。反转必须先降至零、至少保持一个制动周期，再向相反方向升高。
- `STOP` 走受控减速并在停稳后 Brake。可恢复 IMU 降级或未达到锁存门槛的心跳缺失发布运动禁止；`safetyTask` 先通过普通 Coast 覆盖执行器，`controlTask` 随后保持 Coast，不得调用破坏性急停。
- Control 的最终安全门复核、状态机推进与 PWM 提交，和 Safety 的门状态发布与 Coast/Emergency，使用同一个不屏蔽中断的短任务级仲裁边界。两者只能形成“Control 先提交、Safety 后覆盖”或“Safety 先置门、Control 拒绝 Drive”两种顺序。
- 现有 `BspMotor_EmergencyCoastAll()` 会把 PWM 引脚改为 GPIO 低电平、清除 CCER/MOE 并停止 TIM1/TIM9/TIM12，当前没有经过验证的在线重装备路径。因此 `ESTOP`、硬故障和已达到门槛的锁存故障调用该接口后必须通过系统复位恢复；G1 不提供远程清除后重新 ARM。

## UART、遥测与 ADC 边界

- UART4 使用 PC10/PC11、230400 baud，当前 RX 为逐字节中断加 256 字节环形缓冲；UART DMA + IDLE 仍属于 M6，不在 G1 实现。
- 当前 `BspUart_Write()` 是阻塞发送。G1.3 必须增加中断驱动的非阻塞 TX 队列；遥测拥塞可以丢帧并计数，但不得阻塞控制或改变安全行为。
- G1.3 使用最大 64 字节的临时 ASCII 行命令，LF 结束并允许 CR。运动命令必须带单调递增序号；格式错误、溢出、旧序号和越界值不得刷新 500 ms 命令超时。
- 临时试验协议不增加 CRC；显式 ARM、严格解析、序号、硬限幅、斜坡、超时和故障锁存构成 G1 安全边界。正式二进制 CRC 协议仍属于 M5。硬急停锁存不接受远程清除，状态查询必须明确报告需要系统复位。
- 遥测候选周期为 20 ms（50 Hz），在任务上下文格式化。字段至少覆盖硬件时间戳、节拍序号、编码器原始值/增量/累计值、目标/实际 PWM、电池电压、安全状态、唤醒延迟、WCET、丢周期和通信错误计数。
- 编码器每轮计数未实测关闭前，遥测只发布 counts/s，不发布伪精确 RPM。
- 当前 ADC 电池采样由 `monitorTask` 每 500 ms 执行。G1.3 将 ADC 单一所有权移交给 `commTask`，以 20 ms 周期采样；`monitorTask` 只消费已发布的电池快照，禁止两个任务并发操作 HAL ADC。

## 四个提交的独立验收

### G1.0：资源审查

- 仅修改中文文档，不改变固件或 `.ioc` 行为。
- 文档必须明确 TIM6 冲突、TIM7 参数、CubeMX 入口、ISR 边界、任务通知、编码器快照、安全参数、UART 非阻塞发送和 ADC 所有权。
- 通过 `git diff --check` 和跨文档一致性检查后提交。

### G1.1：确定性控制节拍

- CubeMX 可重复生成，且只出现预期 TIM7 变化。
- host 测试覆盖 16 位编码器回绕、64 位累计、无符号 DWT 差值和节拍统计。
- Debug、Release、严格告警和静态分析通过。
- 软件只建立测量能力；P99 抖动、最大抖动、WCET 和零丢周期仍标记为待板测。

### G1.2：MA 安全开环链路

- host 测试覆盖上电未使能、限幅、斜坡、反转过零、STOP、500 ms 超时、ESTOP 锁存且不可远程恢复、可恢复运动禁止、关键故障升级和计数饱和。
- 提交内没有 UART 运动命令入口，因此不能自主驱动电机。
- Debug、Release、严格告警和静态分析通过。
- 软件验收结果：host 5/5，最终 Debug/Release 链接通过，严格告警与静态分析 3/3；两名独立审批者均通过。
- 板上 PWM、Brake/Coast、故障注入、WCET、抖动和栈水位仍未执行，因此状态为“软件完成、待板测”。

### G1.3：UART4 命令与遥测

- host 测试覆盖命令解析、行溢出、数值溢出、旧序号、越界值、非法命令不续命、遥测边界和队列满载。
- UART TX 必须非阻塞，ADC 必须只有一个任务所有者。
- Debug、Release、严格告警和静态分析通过。
- 软件完成后仍需在首次板测中显式 ARM，才允许 MA 低功率运动。

## 板测延期项

- TIM7 连续运行零丢周期，P99 唤醒抖动不超过周期 5%，最大抖动不超过 10%，最坏 WCET 不超过周期 25%。
- MA 输出轴整圈计数、低功率方向符号、电池压降、AT8236 Brake/Coast 和斩波边界。
- M1.3 的 DRDY、量程、ODR、轴映射、时间戳回绕、突变阈值、退避恢复、栈余量和 CPU 占用。

上述项目随 M2 单电机首次集中板测执行；在板测记录形成前，只能宣称相应软件实现和离线验证完成。
