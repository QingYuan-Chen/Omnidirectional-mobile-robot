# 规划审批意见与执行指令 v2

> 本文整合 2026-07-11 Codex 与 Claude 技术对审后获得批准的执行方向。本文管理 M0-M3 的执行约束；`project_roadmap_review.md` 管理完整 M0-M6 路线图。

## 执行顺序

1. M0 仓库基线和规划文档。
2. M0 UART 接收恢复缺陷。
3. M1 可恢复 IMU 故障状态机与数据质量输出。
4. M1 硬件验证。
5. M2 UART4 最小调试台。
6. M2 MA 电机方向标定。
7. M2 单电机 PID 与 host 测试。
8. M3 人机交互状态机。

## M0 执行要求

- 仓库根目录为 `Mobile_robot_lower_level_machine`，默认分支为 `main`。
- 不提交构建目录、IDE 状态、CubeMX 临时输出、厂家参考副本和本地 `参考/` 目录。
- 每个可独立验证的功能在检查通过后形成一次提交。只有调查或测试失败的会话记录到 `progress.md`，不创建空提交或半成品提交。
- 固件变更后必须构建 Debug 和 Release。相关应用文件还要接受严格告警和静态分析。
- CubeMX 生成文件通过 `.ioc` 重新生成或指定的 USER CODE 区修改；应用行为放在 `App/`。

## UART 恢复要求

- 接收完成回调发现 UART 错误时，必须记录错误并安排恢复，不能静默返回。
- 错误处理不能对同一 UART 递归启动两次中断接收。
- 当回调发生时 HAL 接收状态尚未 Ready，`BspUart_Service()` 继续作为任务上下文恢复者。
- 恢复尝试和失败次数通过 `BspUartStats` 可观测。

## IMU 数据链要求

- `raw_sample` 保留 QMI8658A 同步寄存器帧原值，仅用于诊断。
- 机体系 SI 加速度与角速度继续独立输出。ESKF 使用经过有效性门控、但未经普通低通的 SI 样本，避免姿态估计引入额外相位延迟。
- 后续增加独立的滤波加速度和角速度字段，供控制与遥测使用。
- 突变检测先进行样本级拒绝和诊断计数，只有连续失败才升级。阈值可配置，并由板上数据最终确定。
- `sample_age_ms` 由生产者根据 `now_ms - last_good_tick_ms` 计算。

## IMU 故障与退避要求

- 任务存活和传感器健康相互独立。即使传感器处于退避，存活的 IMU 任务也必须继续上报心跳。
- 退避不得使用长阻塞延迟：任务仍每 5-10 ms 运行，通过 `next_retry_tick` 判断何时允许重试 I2C。
- 退避序列为 20、50、100、200、500 ms，最大 500 ms；一次成功读取即复位。
- 单次 I2C 错误、时间戳跳变或加速度观测拒绝只能使受影响数据失效，不能永久锁存机器人。
- 持久传感器故障进入受控停车/降级状态。永久锁存只留给不可恢复的内部错误或明确定义的长时间故障。
- 恢复需要连续稳定、时间戳连续的有效样本，并确认滤波器已初始化且状态有限。
- 丢样、重复、拒绝和 I2C 错误计数必须是真实的饱和计数，不能兼作状态哨兵。

## 单电机速度 PI 要求

- PID 板上调试前，UART4 先提供临时的人类可读调参通道。
- 闭环前必须完成开环方向标定。方向修正在 BSP/配置层完成，不能藏进 PID 算术。
- 当前硬件没有电流 ADC 反馈，禁止用估算值伪装电流反馈或实现伪电流环。
- 速度 PI + 前馈不依赖 HAL，可供四个电机复用，并利用 Cortex-M4F 的 float 运算；可预留位置外环目标接口，但 M2 不实现位置闭环。
- 抗饱和采用条件积分/钳位，仅做输出限幅不够。
- 目标为零时受控减速，停稳后 Brake；500 ms 命令超时复用同一受控停车路径。
- host 测试覆盖换算、饱和、抗积分饱和、斜坡和代表性阶跃序列。

## 人机交互要求

- 写按键行为前，必须依据硬件证据确认按键极性和上下拉。
- 按键、LED 和蜂鸣器使用非阻塞状态机，不增加专用任务。
- 当前一次性启动提示音保留到蜂鸣器 pattern 实现，之后迁移到统一机制，避免两套实现并存。

## M0-M3 明确不做

- 四轮麦轮运动学和四电机联动控制。
- 二进制 X-Protocol 和 ROS2 节点联调。
- IWDG 和 UART DMA + IDLE。
- 在确认电池串数、化学体系和带载电压特性前固定最终低电压阈值。
